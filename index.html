React Hooks · Vue Composition API

Développement Frontend Moderne

React Hooks vs
Vue Composition API

Deux approches modernes pour organiser la logique d'état et de cycle de vie dans des composants maintenables et réutilisables.

React 16.8+ Vue 3.0+

Section 01 — Contexte

Avant la révolution

⚛️

React — Classes Components

Avant les Hooks, React reposait sur des class ES6 héritant de React.Component. La logique était fragmentée entre componentDidMount, componentDidUpdate et componentWillUnmount, rendant la réutilisation difficile et le code long.

💚

Vue — Options API

Vue 2 organisait les composants via des options séparées : data(), methods, computed, watch. Pratique pour les petits composants, mais la logique liée à une même fonctionnalité se retrouvait éparpillée entre plusieurs blocs.

🚧

Le problème commun

Les deux approches souffraient du même défaut : la réutilisation de logique stateful était complexe (mixins, HOC, render props pour React) et le code devenait difficile à lire sur de gros composants. Les Hooks et la Composition API apportent une solution élégante.

Section 02 — React

React Hooks

useState un Hook de React qui permet de créer et gérer une variable dynamique (state) dans un composant.
const [variable, setVariable] = useState(valeurInitiale);
useEffect Gère les effets (eventlistener, DOM) , il permet d'executer le code apres que react affiche le composant. Remplace componentDidMount, componentDidUpdate et componentWillUnmount.
Custom Hooks Fonctions préfixées use* qui encapsulent et réutilisent de la logique stateful entre composants.

Section 03 — Vue

Composition API

ref()

Rend réactif une valeur primitive (string, number, boolean). Accessible via .value.

reactive()

Rend réactif un objet entier. Pratique pour grouper plusieurs états liés.

computed()

une valeur calculée automatiquement à partir d’autres variables réactives

watch()

c’est une fonction de Vue qui permet de surveiller une variable réactive et d’exécuter du code automatiquement lorsqu’elle change.

composables

Fonctions réutilisables (ex : useCounter()) qui encapsulent la logique réactive.

Section 04 — Avantages

Pourquoi adopter ces approches ?

01 🗂️

Meilleure organisation

La logique liée à une même fonctionnalité est regroupée au même endroit, au lieu d'être dispersée dans des méthodes de cycle de vie ou des blocs d'options séparés.

02 ♻️

Réutilisation facilitée

Les hooks et composables sont de simples fonctions JavaScript. Pas de mixins ambigus ni de HOC complexes : la logique stateful se partage naturellement entre composants.

03 🔷

Typage TypeScript simplifié

Les fonctions et variables ordinaires s'intègrent parfaitement avec TypeScript. Fini les types complexes de this dans les classes Vue ou React.

04 🧪

Testabilité améliorée

Les hooks et composables peuvent être testés de façon isolée, sans avoir à faire un composant complet. Les fonctions pures sont bien plus simples à couvrir par des tests unitaires.

Section 05 — Démonstration

Un compteur, deux approches

React — useState
jsx 
function Compteur() {
  const [count, setCount]
    = useState(0);

  return (
    <div>
      <span>{count}</span>
      <button
        onClick={() => setCount(c => c+1)}>
        +
      </button>
      <button
        onClick={() => setCount(c => c-1)}>
        -
      </button>
    </div>
  );
}

Vue — ref()
vue 
<script setup>
import { ref } from 'vue'

const count = ref(0)
const inc = () => count.value++
const dec = () => count.value--
</script>

<template>
  <div>
    <span>{{ count }}</span>
    <button @click="dec">−</button>
    <button @click="inc">+</button>
  </div>
</template>
